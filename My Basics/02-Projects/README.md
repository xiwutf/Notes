# 02-Projects

## 用途

Projects 存放**有明确目标、有截止日期、可交付成果**的工作单元。区别于 Knowledge（永久有效）和 Daily（当日事务）。

每个项目是一个独立的「作战单元」，有启动、推进、交付、复盘全生命周期。

## 使用建议

1. **命名规范**：`PRJ-项目名称`，便于 Dataview 检索与全局搜索
2. **一项目一文件**：使用 [[07-Templates/Project Template|Project Template]] 创建
3. **状态流转**：`规划中 → 进行中 → 暂停 → 完成 → 归档`
4. **项目上限**：同时进行不超过 3 个，聚焦比广度更重要
5. **Weekly Review 对齐**：每周检查项目进度，砍掉不再重要的项目
6. **完成后归档**：移入 `Archive/Projects/`，保留链接不断裂
