# 认知系统说明书

> 第二大脑操作手册 — 如何运行这个系统，而不是被系统运行

---

## 系统哲学

```
输入（Inbox）→ 加工（Thinking/Knowledge）→ 行动（Projects/Daily）→ 输出（Assets）→ 复利
```

这个系统不是笔记软件，是**个人认知操作系统**。Obsidian 是运行环境，目录结构是文件系统，MOC 是索引，AI 是增强引擎。

**五个关键词**：极简 · 长期主义 · 系统思维 · AI 增强 · 工程化

## 一、如何记录思考

### 1.1 三层记录法

| 层级 | 位置 | 时长 | 深度 | 示例 |
|------|------|------|------|------|
| 捕获 | Inbox | 10 秒 | 零 | 快速记录，不做判断 |
| 思考 | Thinking | 15-30 分钟 | 中 | 深度分析，形成观点 |
| 知识 | Knowledge | 30-60 分钟 | 高 | 内化沉淀，可复用 |

**规则**：Inbox 条目必须在 24 小时内处理——升级、转化或删除。

### 1.2 思考笔记三问自检

每写完一条 Thinking 或 Knowledge 笔记，回答：

1. **我真正理解了吗？** — 不是「看懂了」，是「能推导」
2. **我能用自己的话解释吗？** — 「我的理解」区块必须填写
3. **这改变了我的行动吗？** — 如果没有行动项，可能只是收藏

### 1.3 思考触发场景

- 读完后有「啊哈时刻」→ 立即创建 Thinking 笔记
- 项目中遇到架构决策 → 记录决策过程和理由
- 与他人讨论后有新视角 → 捕捉反面论证
- Weekly Review 时有复盘感悟 → 升级为 Thinking

### 1.4 使用 AI 辅助思考

参考 [[06-AI-System/Workflows/AI-工作流索引|WF-01 思考深化]]：
- 用 AI 做反面论证，但不接受 AI 的结论——结论必须是你的
- AI 输出是草稿，「我的理解」才是资产

## 二、如何避免知识收藏癖

### 2.1 收藏癖的症状

- Inbox 超过 20 条未处理
- Knowledge 笔记的「我的理解」区块为空
- 笔记数量增长但 MOC 不更新
- 花更多时间整理笔记而非使用笔记
- Graph View 有很多节点但连接稀疏

### 2.2 预防机制

**机制一：强制「我的理解」**

Knowledge Template 的「我的理解」是必填项。如果填不出来，说明还没理解，保持 `status: 种子`，不要标记为成熟。

**机制二：Inbox 每日清零**

Inbox 是缓冲区，不是仓库。每天处理一次，使用四问法则：

```
这条信息 → 删除？ / 转项目？ / 转知识？ / 转任务？
```

**机制三：输出检验**

一条 Knowledge 笔记从 `种子` → `成长中` → `成熟` 的升级条件：

- 种子：刚创建，有基本内容
- 成长中：链接 ≥ 2 条笔记，「我的理解」已填写
- 成熟：被引用 ≥ 1 次（项目/思考/资产），或已输出为 Asset

**机制四：Weekly Review 审计**

每周检查：
- [ ] 本周新增 Knowledge 数量 ≤ 5（质量 > 数量）
- [ ] 本周是否有 Asset 输出
- [ ] 是否有超过 30 天未更新的「种子」笔记（深化或删除）

### 2.3 收藏 vs 知识 对照

| 收藏 | 知识 |
|------|------|
| 复制粘贴原文 | 用自己的话表述 |
| 无链接 | 链接 ≥ 2 条笔记 |
| 无行动项 | 有明确的应用场景 |
| 永不回顾 | 被项目/输出引用 |
| 越积越多 | 定期升级或归档 |

## 三、如何形成知识网络

### 3.1 双向链接原则

**新建笔记时**：
1. 至少链接 2 条已有笔记
2. 检查被链接的笔记，必要时添加反向链接
3. 更新相关 MOC 索引

**链接类型**：

```
上位概念 ←→ 下位概念     KNW-系统思维 ←→ KNW-长期主义
同领域   ←→ 同领域       KNW-RAG ←→ KNW-AI-Agent架构
知识     ←→ 项目         KNW-数字孪生 ←→ PRJ-xxx
知识     ←→ 思考         KNW-复利效应 ←→ THK-复利思维实践
知识     ←→ 资产         KNW-xxx ←→ AST-xxx
```

### 3.2 MOC 维护节奏

| 频率 | 动作 |
|------|------|
| 每次新建笔记 | 更新对应领域 MOC |
| 每周 | 检查 MOC-Knowledge 和 MOC-Projects |
| 每月 | 检查孤儿笔记（无入链无出链） |
| 每季度 | 重构 MOC 结构，合并/拆分领域 |

### 3.3 知识网络健康指标

```
健康：
  ✅ 平均每条笔记 ≥ 2 个链接
  ✅ MOC 索引与实际笔记同步
  ✅ Graph View 有明显的聚类（按领域）
  ✅ 核心概念枢纽（KNW-系统思维 等）连接度最高

不健康：
  ❌ 大量无链接笔记
  ❌ MOC 与实际内容脱节
  ❌ Graph View 呈「星形」（只有一个中心）
  ❌ 笔记增长但输出为零
```

### 3.4 从网络到输出

```
                    ┌→ Projects（应用）
Knowledge Network ──┤
                    ├→ Thinking（深化）
                    └→ Assets（输出）
```

当某个主题的 Knowledge 笔记 ≥ 3 条且 ≥ 1 条成熟 → 考虑产出 Asset。

## 四、如何长期积累复利资产

### 4.1 短期复利（日）

**Daily Note 连续记录** — 最低成本的复利习惯

```
每天 5 分钟：
  1. 写今日重点（1 条）
  2. 记录 1 个思考/洞察
  3. 更新 1 个项目进展
  4. 处理 Inbox
```

### 4.2 中期复利（月）

**Weekly Review** — 系统的「编译器」

```
每周日 30 分钟：
  1. 回顾 7 天 Daily Notes
  2. 更新项目状态和 MOC
  3. 清空 Inbox
  4. 规划下周 Top 3
  5. 检查是否有可输出的 Knowledge
```

### 4.3 长期复利（年）

**Knowledge → Assets 流水线**

```
Q1: 积累知识（KNW-xxx），建立领域 MOC
Q2: 深化思考（THK-xxx），形成原创观点
Q3: 输出资产（AST-xxx），建立个人 IP
Q4: 复盘迭代，归档过时内容，规划下一年
```

### 4.4 复利资产清单

| 资产类型 | 存放位置 | 复利方式 |
|----------|----------|----------|
| 知识笔记 | 03-Knowledge | 链接越多，理解越深 |
| 思考笔记 | 03-Knowledge/Thinking | 观点越成熟，输出越快 |
| 项目经验 | 02-Projects → Archive | 复盘沉淀为 Knowledge |
| 内容资产 | 05-Assets | 发布后继续引流 |
| Prompt/Workflow | 06-AI-System | 迭代优化，效率倍增 |
| Daily Notes | 08-Daily | 时间轴上的决策记录 |

### 4.5 个人 IP 输出路径

```
1. 选定定位：工业 AI + 数字孪生实践者
2. 持续输入：Knowledge 积累（每周 1-2 条）
3. 深度加工：Thinking 形成观点（每月 1-2 条）
4. 对外输出：Assets 发布（每月 1 篇）
5. 反馈循环：输出反馈 → 新 Thinking → 新 Knowledge
```

## 五、日常操作速查

### 每日（5 分钟 × 2）

```
晨：Dashboard → Daily Note → 写重点 → 处理 Inbox
晚：Daily Note → 复盘 → 记录思考 → 准备明日
```

### 每周（30 分钟）

```
周日：Weekly Review Template → 更新 MOC → 规划下周
```

### 每月（1 小时）

```
- 检查 Knowledge 状态分布
- 清理 Inbox 和 Archive
- 迭代 Prompt/Workflow
- 评估 Asset 输出进度
```

### 每季度（2 小时）

```
- 重构 MOC（如有必要）
- 归档完成项目
- 回顾 Life 目标
- 调整系统结构
```

## 六、系统扩展指南

### 添加新领域

1. 在 `03-Knowledge/` 下创建子目录
2. 创建 `MOC-{领域}.md`
3. 在 [[03-Knowledge/MOC-Knowledge]] 中添加链接
4. 创建 1-2 条种子 Knowledge 笔记

### 添加新工作流

1. 重复执行 3 次以上后固化
2. 在 [[06-AI-System/Workflows/AI-工作流索引]] 中添加
3. 关联对应 Prompt
4. 标注输入/输出目录

### 系统原则

- **先运行，后优化** — 不要一开始就完美
- **减少结构，增加连接** — 链接比文件夹重要
- **输出倒逼输入** — 想发布什么，就学什么
- **AI 是助手，不是替代** — 思考必须是自己完成的
