## 系统哲学

```
输入（Inbox）→ 触发（Insight）→ 知识（KNW）→ 思考（THK）→ 项目（PRJ）→ 输出（AST）→ 复利
                              ↑                                    ↓
                              └────────── SYS 战略层 ←─────────────┘
```

这个系统不是笔记软件，是 **Life OS（人生操作系统）**。Obsidian 是运行环境，目录结构是文件系统，MOC 是索引，SYS 是战略层，AI 是增强引擎。

**设计目标**：AI 开发 · 数字孪生 · 独立开发 · 个人 IP · 财富自由

**六个关键词**：极简 · 长期主义 · 系统思维 · AI 增强 · 工程化 · 行动导向

**四条原则**：

- 少整理，多思考
- 少收藏，多行动
- 少摘抄，多理解
- 少知识，多决策

## 一、笔记分层

### 1.1 七层笔记体系

| 前缀 | 类型 | 粒度 | 回答的问题 | 模板 |
|------|------|------|------------|------|
| BOOK | 读书笔记 | 一本书 | 讲了什么？我如何评价？ | 读书笔记模板 |
| INS | 触动笔记 | 一条触动 | 什么打动我？对项目有何启发？ | 触动笔记模板 |
| KNW | 知识笔记 | 一个概念 | 我理解的可复用概念？ | 知识笔记模板 |
| THK | 思考笔记 | 一个判断 | 我形成了什么观点？ | 思考笔记模板 |
| PRJ | 项目笔记 | 一个交付 | 我要完成什么？ | 项目模板 |
| AST | 输出资产 | 一份输出 | 我对外发布了什么？ | — |
| SYS | 人生主线 | 一个方向 | 这条主线状态如何？ | 人生系统模板 |

### 1.2 触动笔记（INS）是什么

**触动 = 被打动 + 有启发**，不是知识，不是深度思考。

记录：
- 哪句话触动我
- 为什么触动我
- 对我当前项目有什么启发
- 是否改变我的决策

存放：`03-知识库/触动/INS-xxx.md`

### 1.3 知识闭环

```
BOOK → INS → KNW → THK → PRJ → AST
       ↑                    ↓
       └──── SYS 战略层 ────┘
```

**示例**：
- 读《原则》→ BOOK-原则
- 「痛苦+反思=进步」触动我 → INS-痛苦是进化信号
- 提炼可复用概念 → KNW-复盘方法论
- 形成判断「项目失败必须写复盘」→ THK-失败复盘原则
- 推动 → PRJ-数字孪生平台
- 输出 → AST-工业项目复盘实践

### 1.4 四层记录法

| 层级 | 位置 | 时长 | 深度 | 示例 |
|------|------|------|------|------|
| 捕获 | Inbox | 10 秒 | 零 | 快速记录，不做判断 |
| 触发 | Insights | 2-5 分钟 | 低 | 触动句 + 为什么 + 项目启发 |
| 思考 | Thinking | 15-30 分钟 | 中 | 深度分析，形成观点 |
| 知识 | Knowledge | 30-60 分钟 | 高 | 内化沉淀，可复用 |

**规则**：Inbox 条目必须在 24 小时内处理——升级、转化或删除。

### 1.5 三问自检

每写完一条 INS / Thinking / Knowledge 笔记，回答：

1. **我真正理解了吗？** — 不是「看懂了」，是「能推导」
2. **我能用自己的话解释吗？** — 「我的理解」区块必须填写
3. **这改变了我的行动吗？** — 如果没有行动项，可能只是收藏

打开 **[[00-工作台/首页|工作台]]** 或 **[[00-工作台/读书看板|读书看板]]**，点击 **📖 新建读书笔记** → 输入书名 → 自动生成 `BOOK-书名.md` 并打开。

| 📖 新建读书笔记 | `03-知识库/读书/` | `BOOK-书名.md` |
| 💡 新建触动笔记 | `03-知识库/触动/` | `INS-主题.md` |

组件文件：[[00-工作台/快捷按钮|快捷按钮]]（DataviewJS 渲染，需启用 Dataview）

**备选方式**：
- 命令面板 → `Templater: User Script - new_book`

### 2.2 在哪里创建笔记

| 内容 | 目录 | 命名 |
|------|------|------|

| 触动 Insight | `03-知识库/触动/` | `INS-主题.md` |

### 2.3 WF-08 深度阅读工作流

```
开始阅读 → 建 BOOK-xxx
    ↓

    ↓

推动行动   → 链 PRJ-xxx / 写 Daily 任务

    ↓

```

→ 详见 [[06-AI系统/工作流/AI-工作流索引#WF-08 · 深度阅读|WF-08]]

### 2.4 思考触发场景

- 读书遇到触动句 → 立即创建 **INS** 笔记
- 读完后有「啊哈时刻」→ 升级为 **THK** 笔记
- 项目中遇到架构决策 → 记录决策过程和理由
- 与他人讨论后有新视角 → 捕捉反面论证
- Weekly Review 时有复盘感悟 → 升级为 Thinking

### 2.5 使用 AI 辅助

- **ChatGPT 长期上下文**：[[06-AI系统/提示词/系统提示词|Life OS 系统 Prompt]]
- **读书场景**：Prompt 库 →「读书笔记」「Insight 生成」
- **思考场景**：[[06-AI系统/工作流/AI-工作流索引|WF-01 思考深化]]
- AI 输出是草稿，「我的理解」才是资产；结论必须是你的

## 三、如何避免知识收藏癖

### 3.1 收藏癖的症状

- Inbox 超过 20 条未处理
- Knowledge 笔记的「我的理解」区块为空
- BOOK 笔记只有摘抄，没有 INS / 行动项
- INS 笔记 `status: 待关联` 超过 7 天未处理
- 笔记数量增长但 MOC 不更新
- Graph View 有很多节点但连接稀疏

### 3.2 预防机制

**机制一：强制「我的理解」**

Knowledge / Insight Template 的「我的理解」是必填项。填不出来 = 还没理解，保持 `status: 种子` 或 `待关联`。

**机制二：Inbox 每日清零**

```
这条信息 → 删除？ / 转 INS？ / 转项目？ / 转知识？ / 转任务？
```

**机制三：INS 必须关联**

每条 INS 创建后 48 小时内，至少完成一项：
- 链接 KNW 或 THK
- 链接 PRJ 并写启发
- 写入 Daily 行动项

**机制四：输出检验**

| 状态 | 升级条件 |
|------|----------|
| 种子 | 刚创建，有基本内容 |
| 成长中 | 链接 ≥ 2 条笔记，「我的理解」已填写 |
| 成熟 | 被 PRJ/THK/AST 引用 ≥ 1 次 |

**机制五：Weekly Review 审计**

- [ ] 本周新增 KNW ≤ 5（质量 > 数量）
- [ ] 本周是否有 INS → PRJ 的转化
- [ ] 本周是否有 AST 输出
- [ ] 待关联 INS 是否清零

### 3.3 收藏 vs 知识 vs 触发 对照

| 收藏 | Insight | 知识 |
|------|---------|------|
| 复制粘贴原文 | 触动 + 为什么 + 项目启发 | 用自己的话表述 |
| 无链接 | 链接 BOOK + PRJ | 链接 ≥ 2 条笔记 |
| 无行动项 | 有下一步行动 | 有应用场景 |
| 永不回顾 | 48h 内关联或行动 | 被项目/输出引用 |

## 四、如何形成知识网络

### 4.1 双向链接原则

**新建笔记时**：
1. 至少链接 2 条已有笔记
2. 检查被链接的笔记，必要时添加反向链接
3. 更新相关 MOC / SYS 索引

**链接类型**：

```
BOOK  ←→ INS          BOOK-原则 ←→ INS-痛苦是进化信号
INS   ←→ KNW          INS-xxx ←→ KNW-复盘方法论
INS   ←→ PRJ          INS-xxx ←→ PRJ-数字孪生平台
KNW   ←→ THK          KNW-复利效应 ←→ THK-复利思维实践
KNW   ←→ AST          KNW-xxx ←→ AST-xxx
SYS   ←→ PRJ/KNW/INS  SYS-AI事业 ←→ PRJ-xxx
```

### 4.2 MOC 维护节奏

| 频率 | 动作 |
|------|------|
| 每次新建笔记 | 更新对应领域 MOC |
| 每周 | 检查 人生主线、MOC-Knowledge、读书看板 |
| 每月 | 检查孤儿笔记（无入链无出链） |
| 每季度 | 复盘四个 SYS 系统状态 |

### 4.3 知识网络健康指标

```
健康：
  ✅ INS 有 BOOK 来源 + PRJ 或 KNW 去向
  ✅ 平均每条 KNW ≥ 2 个链接
  ✅ SYS 系统关联了活跃 PRJ
  ✅ 读书看板 数据正常更新

不健康：
  ❌ 大量 BOOK 摘抄无 INS
  ❌ INS 堆积 status=待关联
  ❌ SYS 系统空壳无项目
  ❌ 笔记增长但 AST 输出为零
```

### 4.4 从网络到输出

```
                    ┌→ PRJ（应用）
BOOK → INS → KNW ──┤
                    ├→ THK（深化）
                    └→ AST（输出）
                           ↑
                      SYS 战略层统筹
```

当某主题 KNW ≥ 3 条且 ≥ 1 条成熟 → 考虑产出 AST。

## 五、如何长期积累复利资产

### 5.1 短期复利（日）

```
每天 5 分钟：
  1. 打开 Dashboard → 人生主线 确认主线
  2. 写 Daily 今日重点（1 条）
  3. 记录 1 个 INS 或思考
  4. 更新 1 个项目进展
  5. 处理 Inbox
```

### 5.2 中期复利（月）

```
每周日 30 分钟（WF-05 Weekly Review）：
  1. 回顾 7 天 Daily Notes
  2. 更新 PRJ 状态和 SYS 系统
  3. 清空 Inbox + 处理待关联 INS
  4. 规划下周 Top 3
  5. 检查 读书看板
```

### 5.3 长期复利（年）

```
Q1: 积累 KNW + BOOK/INS，建立领域 MOC
Q2: 深化 THK，INS → PRJ 转化
Q3: 输出 AST，建立个人 IP
Q4: 复盘四个 SYS，归档过时内容，规划下一年
```

### 5.4 复利资产清单

| 资产类型 | 存放位置 | 复利方式 |
|----------|----------|----------|
| 读书笔记 | 03-知识库/读书 | 重读时链接新 INS |
| 认知触发 | 03-知识库/触动 | 链接越多，决策越快 |
| 知识笔记 | 03-知识库 | 链接越多，理解越深 |
| 思考笔记 | 03-知识库/思考 | 观点越成熟，输出越快 |
| 战略系统 | 04-生活/系统 | 统筹 PRJ 优先级 |
| 项目经验 | 02-项目 → 归档 | 复盘沉淀为 KNW |
| 内容资产 | 05-输出 | 发布后继续引流 |
| Prompt/Workflow | 06-AI系统 | 迭代优化，效率倍增 |

### 5.5 个人 IP 输出路径

```
1. 定位：工业 AI + 数字孪生实践者（SYS-个人IP系统）
2. 输入：BOOK → INS → KNW（每周 1-2 条）
3. 加工：INS → THK 形成观点（每月 1-2 条）
4. 输出：KNW/THK → AST 发布（每月 1 篇）
5. 反馈：输出反馈 → 新 INS → 新 PRJ
```

## 七、系统扩展指南

### 添加新领域

1. 在 `03-知识库/` 下创建子目录
2. 创建 `MOC-{领域}.md`
3. 在 [[03-知识库/MOC-Knowledge]] 中添加链接
4. 关联到对应 SYS 系统

### 添加新工作流

1. 重复执行 3 次以上后固化
2. 在 [[06-AI系统/工作流/AI-工作流索引]] 中添加
3. 关联对应 Prompt
4. 标注输入/输出目录

### 系统原则

- **先运行，后优化** — 不要一开始就完美
- **减少结构，增加连接** — 链接比文件夹重要
- **输出倒逼输入** — 想发布什么，就学什么
- **INS 驱动行动** — 触动必须流向 PRJ 或 Daily
- **AI 是助手，不是替代** — 思考必须是自己完成的
