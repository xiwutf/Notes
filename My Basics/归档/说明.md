# 归档

## 用途

归档 是**已完成、已失效、已放弃内容的冷存储区**。移入 归档 不是删除——链接保留、历史可查，但不再出现在活跃工作流中。

## 目录结构

```
归档/
├── Projects/      # 已完成/放弃的项目
├── Knowledge/     # 过时但仍有参考价值的知识
├── Assets/        # 已发布/废弃的内容资产
└── Daily/         # 超过 1 年的 Daily Notes（可选）
```

## 使用建议

1. **移入条件**：
   - 项目：状态为「完成」或「放弃」超过 30 天
   - 知识：已被更新版本替代
   - 资产：已发布且无维护需求
2. **不移除链接**：归档 中的文件仍可通过 `[[]]` 访问
3. **Frontmatter 标记**：归档时添加 `archived: true` 和 `archived_date`
4. **Dataview 排除**：Dashboard 查询时过滤 `archived != true`
5. **年度清理**：每年底审视 归档，真正无价值的可以删除

## 归档流程

```
1. 更新笔记 status → 完成/放弃
2. 添加 archived: true
3. 移动到 归档/对应子目录
4. 更新 MOC 索引，移除活跃列表
```
