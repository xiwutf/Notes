# 01-收件箱

## 用途

Inbox 是**零摩擦捕获区**。任何未经加工的想法、链接、截图说明、会议碎片，先扔进来，不做分类、不做评判。

核心原则：**先捕获，后整理**。

## 示例内容

```
- 2026-05-26 看到一篇关于 Agent 架构的文章，值得深读
- 客户提到数字孪生实时同步的痛点
- 想法：个人 IP 可以从「工业 AI 实践者」角度切入
- 链接：https://example.com/digital-twin-paper
```

## 使用建议

1. **每日清零**：Inbox 不应过夜，每天处理一次（建议与 Daily Note 绑定）
2. **处理四问**：这条信息 → 删除 / 转为项目 / 转为知识 / 转为任务？
3. **不超过 20 条**：超过说明处理节奏跟不上，优先清空而非继续堆积
4. **不做深度写作**：Inbox 只写一句话摘要，深度思考去 [[03-知识库/思考|Thinking]] 目录
5. **配合快捷键**：Obsidian 设置 Quick Capture 直达 Inbox

## 处理流向

```
Inbox → 02-项目（可行动的项目）
      → 03-知识库（可复用的知识）
      → 08-日记（当日任务）
      → 归档（无价值，直接归档）
      → 删除
```
