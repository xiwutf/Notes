# AI 工作流索引

> 标准化的 AI 协作流程 — 输入明确、步骤清晰、输出归档

---

## 工作流设计原则

1. **输入标准化**：明确从哪个目录取什么类型的笔记
2. **步骤可重复**：同一流程至少执行 3 次后才固化为 Workflow
3. **输出有归档**：AI 产出必须存入指定目录，经人工审核
4. **Prompt 引用**：每个 Workflow 链接到 [[06-AI系统/提示词/提示词索引|Prompt 库]] 中的具体 Prompt

---

## WF-01 · 思考深化

**场景**：有一个初步想法，需要深度思考

```
触发：Inbox 中出现值得深化的想法
  ↓
输入：Inbox 条目
  ↓
步骤：
  1. 复制「深度思考引导」Prompt
  2. 与 AI 对话 2-3 轮
  3. 使用 [[07-模板/思考笔记模板|Thinking Template]] 创建笔记
  4. 链接至少 2 条已有笔记
  5. 更新 [[03-知识库/思考/MOC-Thinking|MOC-Thinking]]
  ↓
输出：03-知识库/思考/THK-xxx.md
```

---

## WF-02 · 知识内化

**场景**：读了一篇文章/书/课程，需要转化为个人知识

```
触发：阅读完成，Inbox 中有摘要
  ↓
输入：原始材料 + Inbox 摘要
  ↓
步骤：
  1. 使用「知识内化」Prompt 提取核心
  2. 使用 [[07-模板/知识笔记模板|Knowledge Template]] 创建笔记
  3. 填写「我的理解」（必须用自己的话）
  4. 链接相关笔记，更新领域 MOC
  5. 标记 status: 种子
  ↓
输出：03-知识库/{领域}/KNW-xxx.md
```

## WF-03 · 内容创作

**场景**：从成熟知识笔记产出一篇对外内容

```
触发：Knowledge 笔记 status = 成熟，tags 含「可输出」
  ↓
输入：1-3 条相关 Knowledge 笔记
  ↓
步骤：
  1. 使用「文章大纲」Prompt 生成大纲
  2. 人工审校大纲
  3. 使用「技术博客」Prompt 生成初稿
  4. 人工编辑润色
  5. 创建 Assets 笔记，标注来源
  ↓
输出：05-输出/AST-xxx.md
```

## WF-04 · 项目启动

**场景**：Inbox 中的想法值得作为一个项目推进

```
触发：Inbox 条目评估为「可行动的项目」
  ↓
输入：Inbox 条目 + 相关 Knowledge
  ↓
步骤：
  1. 使用「项目规划」Prompt 生成框架
  2. 使用 [[07-模板/项目模板|Project Template]] 创建项目
  3. 设定里程碑和成功标准
  4. 更新 [[02-项目/MOC-Projects|MOC-Projects]]
  5. 在 Daily Note 中添加项目任务
  ↓
输出：02-项目/PRJ-xxx.md
```

## WF-07 · 读书笔记

**场景**：读一本书，需要系统记录并沉淀为个人知识

```
触发：开始读一本新书 / 读完一章 / 全书读完
  ↓
输入：书籍信息 + 章节内容或 Inbox 摘要
  ↓
步骤：
  1. 使用 [[07-模板/读书笔记模板|Reading Template]] 创建 BOOK-xxx 笔记
  2. 阅读过程中：Inbox 快速捕获，章节笔记写入 BOOK 笔记
  3. 使用「读书笔记」Prompt 辅助整理章节摘要与核心收获
  4. 可复用概念 → 拆出 KNW-xxx（[[07-模板/知识笔记模板|Knowledge Template]]）
  5. 啊哈时刻 → 升级 THK-xxx（[[07-模板/思考笔记模板|Thinking Template]]）
  6. 全书读完后：填写「全书一句话」「我的评价」「行动」
  7. 更新 BOOK 笔记的「提炼的知识」和「啊哈时刻」链接表
  ↓
输出：03-知识库/读书/BOOK-xxx.md + 若干 KNW/THK 笔记
```

> 轻量阅读用 WF-07；需要完整知识闭环时用 [[#WF-08 · 深度阅读|WF-08]]。

## WF-08 · 深度阅读

**场景**：深度阅读一本书/长文，从触达行动形成完整闭环

```
触发：开始阅读 / 遇到触动句 / 章节读完 / 全书读完
  ↓
输入：书籍信息 + 触动句 + Inbox 摘要
  ↓
步骤：
  1. 使用 [[07-模板/读书笔记模板|Reading Template]] 创建或更新 BOOK-xxx
  2. 记录「触动我的句子」→ 每条升级为 INS-xxx（[[07-模板/触动笔记模板|Insight Template]]）
  3. 使用「Insight 生成」Prompt，获取 KNW / THK / PRJ 链接建议
  4. 可复用概念 → 拆出 KNW-xxx（[[07-模板/知识笔记模板|Knowledge Template]]）
  5. 形成判断 → 升级 THK-xxx（[[07-模板/思考笔记模板|Thinking Template]]）
  6. 推动行动 → 链接或更新 PRJ-xxx，写入 Daily 任务
  7. 成熟输出 → 从 KNW/THK 沉淀 AST-xxx
  8. 更新 BOOK 的 Insight / 项目关系 / 决策影响 区块
  9. 关联对应 SYS 系统（AI事业 / 财富 / IP / 健康）
  ↓
输出：BOOK + INS + KNW + THK + PRJ + AST（按需）
```

**知识闭环**：

```
BOOK → INS → KNW → THK → PRJ → AST
       ↑                    ↓
       └────── SYS 战略层 ──┘
```

## WF-05 · 每周复盘

**场景**：Weekly Review

```
触发：每周日
  ↓
输入：本周 Daily Notes + 项目状态 + Inbox
  ↓
步骤：
  1. 使用 [[07-模板/每周复盘模板|Weekly Review Template]]
  2. 使用「Weekly Review」Prompt 辅助分析
  3. 更新所有 MOC 索引
  4. 清空 Inbox
  5. 规划下周 Top 3
  ↓
输出：08-日记/复盘/YYYY-Www.md
```

## WF-06 · 个人 RAG 查询

**场景**：基于个人知识库回答问题（需外部工具）

```
触发：需要回忆/查找已有知识
  ↓
输入：自然语言问题
  ↓
步骤：
  1. 使用 Obsidian 搜索 / Dataview 查询
  2. （进阶）向量化笔记库，使用 RAG 检索
  3. 引用相关笔记链接
  ↓
输出：答案 + 来源笔记链接
```
