# 03-知识库

## 用途

Knowledge 是认知系统的**长期复利引擎**。存放经过思考、验证、可复用的知识——不是收藏，是**内化后的认知资产**。

按领域组织，支持双向链接，形成可生长的知识网络。

## 目录结构

```
03-知识库/
├── AI/              # AI 开发、Agent、LLM 相关
├── Books/           # 读书笔记（BOOK-）
├── Insights/        # 认知触发点（INS-）
├── Digital-Twin/    # 工业数字孪生
├── Engineering/     # 工程方法论、架构、工具
├── Thinking/        # 深度思考、洞察、原创观点
└── MOC-Knowledge.md # 知识网络总索引
```

## 笔记分层（Life OS v2.0）

| 前缀 | 类型 | 回答的问题 |
|------|------|------------|
| BOOK | 读书 | 这本书讲了什么？我如何评价？ |
| INS | 触发 | 什么触动了我？对项目有何启发？ |
| KNW | 知识 | 我理解了什么可复用概念？ |
| THK | 思考 | 我形成了什么判断？ |

**闭环**：BOOK → INS → KNW → THK → PRJ → AST

## 使用建议

1. **命名规范**：
   - 知识笔记：`KNW-主题`
   - 思考笔记：`THK-主题`
2. **每条知识必答三问**（见 [[认知系统说明书#如何记录思考]]）
3. **双向链接**：新建笔记时至少链接 2 条已有笔记
4. **MOC 维护**：新领域知识超过 5 条时，更新对应 MOC
5. **拒绝收藏癖**：没有「我的理解」的笔记不算知识，只是书签
6. **输出导向**：定期从 Knowledge 中选取素材，转化为 [[05-输出|Assets]] 内容
